Git

Explanations

Branches
Branches in Git are ways that devlopers can test and develop code without affecting the main part of the project.
Merge
Merging in terms of Git is a way of assimilating branches into a single branch, adding the target branch's code to the current branch's.
Commit
One can think of Commiting in git as saving, setting a 'save point' of sorts allowing a user to go back to this set of code if they run into any bugs or want to change somthing.
Tags
Tags in Git can be seen as more specific branches. While branches can be changed, Tags cannot.
Repositories
Repositories can be seen as folders. They track all changes made to projects, letting you look back at the history of your project.
