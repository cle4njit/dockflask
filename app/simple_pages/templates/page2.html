Explanations

Containers
A Docker container is an executable pack of software with added dependencies that allow it to be run in any environment.
Images
Docker Images are files that are used to execute code found in a Docker Container.

Docker Commands Known
docker run- | creates a container from an image and starts the container with the commands given.
docker build- | creates an image from the code given.
docker images- | checks the current images on our local machine.
docker ps- | this lists all of the running containers on the local machine.
docker stop- | this stops any container specified with their id/name.
docker restart- | this stops and starts the specified container.
docker rm- | this removes the specified container.
docker pull- | this pulls images from the docker repository.
Explanation of starting Project and Accessing Container
In order to start the Project, one first needs the code. To get the code, you can use docker pull cle4/docker_flask:master to pull the needed code. once you have the code, you can use docker run -p 80:80 cle4/docker_flask to construct a container from the image, and set it to the localhost port, allowing you to see what the container builds.
