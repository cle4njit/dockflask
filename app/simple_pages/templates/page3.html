Simple Pages creation & testing
simple pages was created by pulling the blueprint formula from flask, which when used in tandem with the html pages created in the templates folder, and supporting css and images in the static folder allowed for me to check that everything was working as intended when ran.
Project Outline
  • dockflask <-- root of the project
    • .github <-- contains the workflows folder for GitHub to use
      • workflows <-- integration for github workflows to run tests on every push to GitHub
    • app <-- contains html and css files as well as simple_pages to run the application.
      • simple_pages <-- contains the html files and tests for them.
        • templates <-- contains all html files.
      • static <-- contains the css and image files required for the html files.
        • css <-- contains all css files for the project
        • images <-- contains all images required for the project
    • calculator <-- contains calculation and operation files for the calculator.
      • calculation <-- contains the source code to run addition, subtracton and multiplication constructor methods.
      • operations <-- contains the static methods which call on the constructor methods to output desired results.
    • .tests <-- contains all the test python files from calculator to homepage tests, which ensure everything is working as intended.
    • .dockerignore <-- tells Docker what command to ignore within the code.
    • .gitignore <-- tells git what commands to ignore in the code.
    • docker-compose.yml <-- contains the settings to develop on your local machine
    • Dockerfile <-- contains all the commands required to build images.
    • heroku.yml <-- allows one to construct docker images on heroku.
    • pytest.ini <-- reroutes testpaths to tests and filterwarnings to ignore
    • readme.md <-- comments and instructions to any viewers of the repository / file.
    • requirements.txt <-- tells the developer what requirements are needed to work on the code.
